9 /* Base configuration variable resolver.
10  *
11  * Theoria configuration files can contain variables of the form _$variable_ ('first variables') and
12  * _$$variable_ ('last variables ') . Resolvers determine how a variable is bound to a value.
13  * Resolvers can be chained together in a sequence so that the value of a variable can be
14  * determined by walking a list of different resolvers.
15  *
16  * _First Variables_ ($var) are bound by the first resolver that recognizes the variable
17  * _Last Variables_ ($$var) are bound by the last resolver that recognizes the variable
18  *
19  * Example ConfigVariableResolvers are: <CmdLineResolver>, <EnvVarResolver>, <ConfigVariableResolver>, <TomlResolver>
20  *
21  * Say you want your project to be configurable from both the command line and environment and you want the command line
22  * to take precedence. This means if a variable is give a value on the command line ignore any value in the env.
23  *
24  * For this you should use: CmdLineResolver -> EnvVarResolver and variables of the form $va1, $var2 and so on.
25  * But suppose for some specific variable you don't want the user to change it if it is defined in the env.
26  * You can still use the same resolver chain but make this a _last variable_ $$specialVar
27  *
28  * A general guideline is to construct your resolver chain such that most variables are $first and only used
29  * $$last syntax for special exceptions. Many projects can probably avoid $$last variables all together.
30  *
31  * Theoria will output a representation of your config file at runtime that shows how variables were resolved.
32  * You can also run a project using: _theoria --resolve-only_ to see the same with out executing the code
33  */
